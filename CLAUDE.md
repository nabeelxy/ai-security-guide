# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a Git-based **documentation knowledge base** on AI security — no build system, tests, or executable code. All content is markdown. The repo has ~86 markdown files across ~48 directories and ~800+ commits of active maintenance.

## Content Structure

Four top-level sections:

- `security_for_ai/` — Attacks and defenses on AI models (jailbreaking, prompt injection, poisoning, adversarial ML, RAG security, agent security, ML privacy, model supply chain, red teaming, benchmarks, MCP security, vibe coding)
- `ai_for_security/` — Using AI to enhance cybersecurity (web/network/IoT/cloud/email/browser security, deepfakes, disinformation, binary analysis, threat intelligence, foundation security models)
- `ai_product_security/` — Engineering security into AI systems (security for AI systems, designing AI security detectors)
- `ai_for_offensive_security/` — AI for ethical hacking/red teaming
- `ai_powered_attacks/` — Real-world AI-assisted attack tracking

## Document Conventions

Each section follows a consistent pattern:
- `README.md` at each level serves as the index/overview with links and curated publication tables
- `reviews/` subdirectories contain deep-dive summaries of individual research papers
- Publication tables use columns: Date | Title | Venue | Link | Tags
- Cross-linking between related topics is preferred

## LLM Wiki Pattern

This repo implements the pattern described in `LLM_WIKI_IDEA.md`. The three layers are:
1. **Raw sources** — immutable source documents (never modified by the LLM)
2. **The wiki** — this repo's markdown files (the LLM writes/updates these)
3. **The schema** — this CLAUDE.md file (defines conventions and workflows)

### Ingest Workflow
When adding a new source:
1. Read the source and discuss key takeaways
2. Write or update a summary page in the appropriate section
3. Update the section `README.md` publication table
4. Update related concept/entity pages with cross-references
5. Note contradictions with existing claims if any

### Query Workflow
When answering questions about content:
1. Check the relevant section `README.md` first to locate relevant pages
2. Read the specific topic pages and any linked `reviews/`
3. Synthesize with citations to specific files
4. Valuable analyses can be filed back as new pages

### Lint Workflow
Periodically health-check the wiki by looking for:
- Contradictions between pages
- Stale claims superseded by newer sources
- Orphan pages with no inbound links
- Important concepts mentioned but lacking their own page
- Missing cross-references

## Docusaurus Site (`site/`)

The wiki is deployed as a Docusaurus 3.10.0 static site from the `site/` subdirectory.

**Development:**
```bash
cd site
npm install --legacy-peer-deps   # first time
npm start                        # dev server at http://localhost:3000
npm run build                    # production build → site/build/
npm run serve                    # preview production build
```

**Key quirk:** Docusaurus 3.10.0 + webpack 5.106.x breaks webpackbar 5/6.x. The `package.json` overrides `webpackbar` to `7.0.0` which fixes the incompatibility. Do not remove this override.

**Docs source:** The docs plugin reads from `..` (the repo root), not a `docs/` subdirectory. All section content lives in the root-level directories (`security_for_ai/`, `ai_for_security/`, etc.).

**Sidebar:** Fully autogenerated via `sidebars.js`. Category order and labels are controlled by `_category_.json` files in each directory. All top-level and most subsection directories have `_category_.json` files.

**Deployment:** GitHub Actions (`.github/workflows/deploy-docs.yml`) builds and deploys to GitHub Pages on every push to `main`. The live URL is `https://nabeelxy.github.io/ai-security-guide/`.

**Search:** `@easyops-cn/docusaurus-search-local` with a local search index (no Algolia needed). Search index is built at compile time.

**Adding new content to the site:** Just add/edit `.md` files in the section directories. The site rebuilds automatically on push. No Docusaurus-specific configuration needed for new markdown files.

## Working in This Repo

Since this is documentation-only:
- Use `grep`/`Grep` to search content across files
- Use `git log --follow` to trace history of specific files
- When adding content, match the style/format of existing files in that section
- Publication tables in README files are sorted by date (most recent first)
- Academic venues commonly referenced: USENIX Security, IEEE S&P, NDSS, NeurIPS, ICLR, CCS, NDSS, ACM CCS
